> A multilingual, multi-agent farming advisory application powered by Google ADK, Gemini, MCP, and Streamlit.

## Overview

AgriSense AI turns a farmer's location, crop, and question into a structured agricultural advisory. Instead of producing one undifferentiated response, the application runs specialized agents in sequence for weather, crop health, and market guidance, then combines their findings into a final report.

It is designed for farmers, agricultural advisors, students, and developers exploring practical multi-agent AI. The guided Streamlit interface keeps each recommendation reviewable and makes the completed report easy to retain as a Markdown file.

> [!IMPORTANT]
> AgriSense AI provides AI-generated guidance, not a substitute for a local agronomist, plant pathologist, market authority, or official severe-weather service.

## Features

- 🌦️ Live seven-day forecasts from Open-Meteo, exposed to the Weather Agent through MCP
- 🌡️ Weather alerts and suggested planting or harvesting windows
- 🩺 Text-based crop symptom and farming-question analysis
- 📊 Crop market guidance, selling timing, and practical selling tips generated by Gemini
- 🤖 Four specialized agents orchestrated in a sequential workflow
- 💡 Human confirmation checkpoints after weather, crop, and market reports
- 🌍 Interface and advisory output support for English, Urdu, Hindi, Punjabi, Arabic, and Spanish
- ✅ Normalization and validation for location, crop, and problem inputs
- 📋 Consolidated, farmer-friendly final advisory with a summary checklist
- 📥 Downloadable Markdown report
- 🎨 Responsive Streamlit interface with a session-only sign-in gate

## System Architecture

```text
Farmer input: location + crop + question
                    |
                    v
         Normalize and validate input
                    |
                    v
     Weather Agent <---- MCP ----> Open-Meteo APIs
                    |
             Farmer confirms
                    v
          Crop Doctor Agent
                    |
             Farmer confirms
                    v
          Market Price Agent
                    |
             Farmer confirms
                    v
            Advisory Agent
                    |
                    v
    Final report + Markdown download
```

The Streamlit application controls the workflow with session state. Each agent is a Google ADK `LlmAgent` using `gemini-2.5-flash`; agent-specific instructions live beside the agent in `SKILL.md`. The Weather Agent alone uses an external tool: a local MCP server that geocodes the location and retrieves forecast data from Open-Meteo.

If the weather service cannot resolve a location or return forecast data, the MCP tool returns a clearly labelled simulated forecast so the workflow can continue.

## Multi-Agent Architecture

| Agent | Purpose | Inputs | Responsibilities | Output |
| --- | --- | --- | --- | --- |
| **Weather Agent** | Convert forecast data into farm guidance | Farm location, crop, selected language | Calls the MCP weather tool; evaluates temperature, rain, wind, alerts, and fieldwork windows | Concise seven-day weather advisory |
| **Crop Doctor Agent** | Address crop health or general farming questions | Location, crop, reported symptoms/question, selected language | Interprets the text description and suggests practical actions using Gemini's model knowledge | Crop diagnosis or farming guidance |
| **Market Price Agent** | Provide selling guidance | Crop, location, selected language | Generates price-oriented market commentary, selling timing, and three selling tips using Gemini's model knowledge | Market and selling advisory |
| **Advisory Agent** | Synthesize the specialist results | Original inputs and all three agent reports | Produces a concise, accessible report with bullets and a checklist | Unified final advisory |

> [!NOTE]
> Crop diagnosis and market guidance do not currently query specialist databases or live market feeds. Treat diagnoses as preliminary and verify prices with a current local source before making decisions.

## Installation

### Prerequisites

- Python 3.10 or newer
- A [Google AI Studio](https://aistudio.google.com/app/apikey) Gemini API key
- Git

### 1. Clone the repository

```bash
git clone https://github.com/ridaamir85/agrisense-ai-agent.git
cd agrisense-ai-agent
```

### 2. Create and activate a virtual environment

```bash
python -m venv .venv
```

Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
```

macOS or Linux:

```bash
source .venv/bin/activate
```

### 3. Install dependencies

```bash
python -m pip install --upgrade pip
pip install -r requirements.txt
```

### 4. Configure credentials

Create a `.env` file in the repository root:

```dotenv
GEMINI_API_KEY=your_gemini_api_key
# Optional; retained by the UI for configuration compatibility
OPENWEATHER_API_KEY=your_openweather_api_key
```

The current weather implementation uses Open-Meteo and does not require an OpenWeather key. You may also enter either key in the application's sidebar; password fields mask their display.

### 5. Run the application

```bash
streamlit run app.py
```

Open the local URL printed by Streamlit, typically `http://localhost:8501`.

## Usage

1. Sign in with a syntactically valid username or email and password. This is a session-only UI gate, not persistent account authentication.
2. Choose an advisory language from the sidebar.
3. Add a Gemini API key if one was not loaded from `.env`.
4. Enter the farm location, select a crop, and describe the issue or question in 15–500 meaningful characters.
5. Start the analysis and review the Weather Agent's report.
6. Confirm each checkpoint to continue through Crop Doctor and Market Price analysis.
7. Compile the unified advisory and download it as a `.md` file.

Changing the language after reports have been generated triggers translation of the completed specialist reports.

## Security and Privacy

- **Human in the loop:** the workflow pauses after every specialist report and requires explicit confirmation before continuing.
- **Credential handling:** `.env` is ignored by Git, sidebar key fields are masked, and keys are placed in process environment variables for the active app process. Never commit credentials.
- **Input controls:** text is Unicode-normalized, control characters are removed, lengths are bounded, crops come from an allow-list, and trivial input is rejected.
- **Session privacy:** farm inputs, credentials, and generated reports are not written to an application database by the current code; Streamlit keeps workflow data in session memory.
- **External processing:** prompts are sent to Google's Gemini service, and the farm location is sent to Open-Meteo. Review those providers' privacy terms before entering sensitive information.
- **Authentication limitation:** the current sign-in screen does not verify credentials against a user store and should not be treated as production access control.
